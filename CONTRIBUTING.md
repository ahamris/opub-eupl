# Bijdragen aan oPub

Bedankt voor uw interesse in het bijdragen aan oPub! Dit document beschrijft hoe u kunt bijdragen.

## Gedragscode

Wij verwachten dat alle bijdragers respectvol met elkaar omgaan. Discriminatie, intimidatie en onprofessioneel gedrag worden niet getolereerd.

## Hoe bijdragen?

### Bugs melden

1. Controleer of de bug al gemeld is in [Issues](https://github.com/ahamris/opub-eupl/issues)
2. Maak een nieuw issue aan met:
   - Duidelijke titel
   - Stappen om het probleem te reproduceren
   - Verwacht gedrag vs. werkelijk gedrag
   - Screenshots indien relevant

### Feature requests

1. Controleer de [ROADMAP.md](ROADMAP.md) voor geplande features
2. Maak een issue aan met het label `enhancement`
3. Beschrijf de gewenste functionaliteit en het verwachte gebruik

### Code bijdragen

1. Fork de repository
2. Maak een feature branch: `git checkout -b feature/mijn-feature`
3. Maak uw wijzigingen
4. Schrijf of update tests waar nodig
5. Commit met een duidelijk bericht (zie conventie hieronder)
6. Push naar uw fork: `git push origin feature/mijn-feature`
7. Maak een Pull Request aan

### Commit conventie

```
type(scope): korte beschrijving

- Detail 1
- Detail 2
```

Types: `feat`, `fix`, `docs`, `style`, `refactor`, `test`, `chore`

### Code stijl

- **PHP**: PSR-12 (Laravel Pint)
- **TypeScript/React**: ESLint + Prettier
- **CSS**: Tailwind CSS utility classes
