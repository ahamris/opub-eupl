## 2. Installation & Setup

### 2.1 Current Installation

Font Awesome is already installed in:
```
vendor/fontawesome/
```

### 2.2 Recommended Setup (CSS Method)

**Option 1: Link CSS in Blade Layout**

Add to your main layout file (e.g., `resources/views/layouts/app.blade.php`):

```blade
<!DOCTYPE html>
<html lang="nl">
<head>
    <meta charset="utf-8">
    <meta name="viewport" content="width=device-width, initial-scale=1">
    <title>@yield('title', 'Open Overheid')</title>
    
    @vite(['resources/css/app.css', 'resources/js/app.js'])
    
    {{-- Font Awesome CSS --}}
    <link rel="stylesheet" href="{{ asset('vendor/fontawesome/css/all.min.css') }}">
</head>
<body>
    @yield('content')
</body>
</html>
```

**Option 2: Import in CSS (Vite/Tailwind)**

Add to `resources/css/app.css`:

```css
@import 'tailwindcss';

/* Font Awesome */
@import '../../vendor/fontawesome/css/all.min.css';

@source '../../vendor/laravel/framework/src/Illuminate/Pagination/resources/views/*.blade.php';
@source '../../storage/framework/views/*.php';
@source '../**/*.blade.php';
@source '../**/*.js';
```

**Option 3: Copy to Public Directory**

```bash
# Copy Font Awesome to public directory
cp -r vendor/fontawesome public/vendor/fontawesome
```

Then reference:
```blade
<link rel="stylesheet" href="{{ asset('vendor/fontawesome/css/all.min.css') }}">
```

### 2.3 JavaScript Method (SVG with JS)

If you prefer SVG icons loaded via JavaScript:

```blade
<script src="{{ asset('vendor/fontawesome/js/all.min.js') }}" defer></script>
```

**Note**: CSS method is recommended for better performance and SEO.

## 3. Implementation Methods

### 3.1 CSS Method (Recommended)

**Pros:**
- Better performance (fonts cached)
- Works without JavaScript
- Better for SEO
- Smaller initial load

**Cons:**
- Requires font files
- Less flexible styling

**Usage:**
```html
<i class="fas fa-search"></i>
<i class="far fa-heart"></i>
<i class="fab fa-github"></i>
```

### 3.2 SVG with JS Method

**Pros:**
- More flexible styling
- Better for animations
- No font files needed

**Cons:**
- Requires JavaScript
- Larger JavaScript bundle
- Not SEO-friendly

**Usage:**
```html
<i class="fas fa-search"></i>
```

### 3.3 Inline SVG Method

**Pros:**
- Full control over styling
- No external dependencies
- Best for customizations

**Cons:**
- More verbose HTML
- Harder to maintain

**Usage:**
```html
<svg class="w-5 h-5" fill="currentColor" viewBox="0 0 512 512">
  <path d="M416 208c0 45.9-14.9 88.3-40 122.7L502.6 457.4c12.5 12.5 12.5 32.8 0 45.3s-32.8 12.5-45.3 0L330.7 376c-34.4 25.2-76.8 40-122.7 40C93.1 416 0 322.9 0 208S93.1 0 208 0S416 93.1 416 208zM208 352a144 144 0 1 0 0-288 144 144 0 1 0 0 288z"/>
</svg>
```

## 4. Icon Families

### 4.1 Solid Icons (`fas` or `fa-solid`)

**Class Prefix**: `fas` or `fa-solid`

**Usage:**
```html
<i class="fas fa-home"></i>
<i class="fa-solid fa-user"></i>
```

**Characteristics:**
- Filled/solid appearance
- Most common style
- 1,392 icons available
- Font weight: 900

### 4.2 Regular Icons (`far` or `fa-regular`)

**Class Prefix**: `far` or `fa-regular`

**Usage:**
```html
<i class="far fa-heart"></i>
<i class="fa-regular fa-star"></i>
```

**Characteristics:**
- Outlined appearance
- 163 icons available
- Font weight: 400
- Not all icons have regular variant

### 4.3 Brand Icons (`fab` or `fa-brands`)

**Class Prefix**: `fab` or `fa-brands`

**Usage:**
```html
<i class="fab fa-github"></i>
<i class="fa-brands fa-twitter"></i>
```

**Characteristics:**
- Brand logos
- 490 icons available
- Font weight: 400
- Use only to represent the actual brand

```js

  },
}
```

## 7. Accessibility

### 7.1 Decorative Icons

Icons that are purely decorative should be hidden from screen readers:

```html
<i class="fas fa-star" aria-hidden="true"></i>
```

### 7.2 Semantic Icons

Icons that convey meaning should have accessible text:

```html
<!-- Icon with visible text -->
<button>
    <i class="fas fa-search" aria-hidden="true"></i>
    <span>Search</span>
</button>

<!-- Icon-only button with screen reader text -->
<button aria-label="Close dialog">
    <i class="fas fa-times" aria-hidden="true"></i>
    <span class="sr-only">Close</span>
</button>

<!-- Icon in link -->
<a href="/settings" aria-label="Settings">
    <i class="fas fa-cog" aria-hidden="true"></i>
    <span class="sr-only">Settings</span>
</a>
```

### 7.3 Focus States

Ensure icons in interactive elements have proper focus indicators:

```html
<button class="focus:outline-2 focus:outline-primary focus:outline-offset-2">
    <i class="fas fa-heart" aria-hidden="true"></i>
    <span class="sr-only">Like</span>
</button>
```

## 9. AI Tool Guidelines

### 9.1 Code Generation Rules

When generating HTML with Font Awesome icons, AI tools should:

1. **Always use proper icon classes**:
   ```html
   <!-- Good -->
   <i class="fas fa-search"></i>
   
   <!-- Bad -->
   <i class="fa fa-search"></i>  <!-- Missing style prefix -->
   ```

2. **Include accessibility attributes**:
   ```html
   <!-- Decorative icon -->
   <i class="fas fa-star" aria-hidden="true"></i>
   
   <!-- Semantic icon -->
   <button aria-label="Search">
       <i class="fas fa-search" aria-hidden="true"></i>
       <span class="sr-only">Search</span>
   </button>
   ```

3. **Use appropriate icon families**:
   ```html
   <!-- Solid (most common) -->
   <i class="fas fa-home"></i>
   
   <!-- Regular (outlined) -->
   <i class="far fa-heart"></i>
   
   <!-- Brands (logos only) -->
   <i class="fab fa-github"></i>
   ```

4. **Combine with Tailwind CSS classes**:
   ```html
   <i class="fas fa-search text-primary text-xl"></i>
   ```

5. **Use proper sizing**:
   ```html
   <!-- With Font Awesome classes -->
   <i class="fas fa-home fa-2x"></i>
   
   <!-- With Tailwind classes (preferred) -->
   <i class="fas fa-home text-2xl"></i>
   ```

6. **Include icons in interactive elements properly**:
   ```html
   <button class="flex items-center gap-2 
                  focus:outline-2 focus:outline-primary focus:outline-offset-2">
       <i class="fas fa-search" aria-hidden="true"></i>
       <span>Search</span>
   </button>
   ```

### 9.2 Common Patterns

**Search Input:**
```html
<div class="relative">
    <i class="fas fa-search absolute left-4 top-1/2 -translate-y-1/2

```

**Icon Button:**

```html
        class="text-on-surface-variant hover:bg-surface-variant
               focus:outline-2 focus:outline-primary focus:outline-offset-2

        aria-label="Close">
    <i class="fas fa-times" aria-hidden="true"></i>

```

**Status Indicator:**
```html

    <i class="fas fa-check-circle text-green-500" aria-hidden="true"></i>
    <span>Success</span>

```


### 9.3 Validation Checklist

Before finalizing code with Font Awesome icons:

- [ ] Uses correct icon family prefix (`fas`, `far`, or `fab`)
- [ ] Includes `aria-hidden="true"` for decorative icons
- [ ] Includes accessible text for semantic icons
- [ ] Uses proper sizing (Tailwind classes preferred)
- [ ] Icons in buttons have proper focus states
- [ ] Icon-only buttons have `aria-label` or screen reader text
- [ ] Icons are properly aligned with text/content
