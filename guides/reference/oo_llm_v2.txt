````txt
# oo_llm.txt
Open Overheid – Zoek API
Reference Architecture & JSON Schema (framework-agnostic)

------------------------------------------------------------
1. GOAL
------------------------------------------------------------

We integrate with the Open Overheid “openbaarmakingen” search API.

This file defines:

- The endpoints we use
- All supported query features (filters + pagination)
- How to get a detailed document
- JSON Schemas for:
  - search result items
  - detail document responses

This file is meant for humans AND AI assistants (Cursor, Copilot, etc.) so they generate correct code and queries and do NOT invent parameters or shapes.


------------------------------------------------------------
2. ENDPOINTS
------------------------------------------------------------

Base search endpoint:

  GET https://open.overheid.nl/overheid/openbaarmakingen/api/v0/zoek

Detail endpoint (single document):

  GET https://open.overheid.nl/overheid/openbaarmakingen/api/v0/zoek/{id}

Example detail URL:

  https://open.overheid.nl/overheid/openbaarmakingen/api/v0/zoek/oep-0b03c64e8b44a7b0466a809054860a2c77fb628b

Where `{id}` is the document identifier returned by the search API.
In search results this often appears as:

  document.id = "oep-..._1"
  document.pid = "https://open.overheid.nl/documenten/oep-..."

For the detail endpoint we use the **base ID** without suffix:
- Either strip trailing `_1`
- Or extract the last path segment from `pid`


------------------------------------------------------------
3. SEARCH QUERY PARAMETERS
------------------------------------------------------------

Core parameters (always allowed):

- zoektekst           : string (may be empty)
- start               : integer, 0-based offset
- aantalResultaten    : integer, MUST be 10, 20 or 50

Filters (all optional, single-valued):

- publicatiedatumVan  : string, format DD-MM-YYYY
- publicatiedatumTot  : string, format DD-MM-YYYY
- documentsoort       : string, e.g. "beslisnota"
- informatiecategorie : string, e.g. "adviezen"
- thema               : string, e.g. "afval"
- organisatie         : string, e.g. "1Stroom"

Pagination rule:

  start = (page - 1) * aantalResultaten

There is NO “page” parameter on the API itself. Always use `start`.


------------------------------------------------------------
4. USAGE PATTERNS
------------------------------------------------------------

SEARCH FLOW
1. User enters zoektekst and optional filters.
2. Client calculates `start` from page & page size.
3. Client calls /zoek with:
   - zoektekst
   - start
   - aantalResultaten
   - plus any chosen filters
4. Client processes each hit in the result array/object.

DETAIL FLOW
1. User clicks a document.
2. Client takes the document ID (from search result).
3. Client calls /zoek/{id}.
4. Client uses detail JSON to show all metadata and file links.


------------------------------------------------------------
5. JSON SCHEMA – SEARCH RESULTS
------------------------------------------------------------

NOTE:
- The exact top-level wrapper of the search response may evolve.
- The important reusable unit is a single “hit” object.
- Many APIs wrap it like: { "results": [ Hit, Hit, ... ], ... }
- Others return just an array; you can adapt easily.

We define a generic “hit item” schema `OpenOverheidSearchHit` that models what we saw in the live API.

  "required": ["document"],
  "properties": {
    "document": {
      "type": "object",
      "description": "Basic metadata for a document in the search result.",
      "required": [
        "id",
        "pid",
        "titel",
        "openbaarmakingsdatum",
        "weblocatie",
        "aanbieder"
      ],
      "properties": {
        "id": {
          "type": "string",
          "description": "Internal search hit ID, often 'oep-..._1'. Use pid-derived base ID for detail calls."
        },
        "pid": {
          "type": "string",
          "format": "uri",
          "description": "Persistent identifier; last path segment usually matches the base detail ID."
        },
        "titel": {
          "type": "string",
          "description": "Human-readable title."
        },
        "omschrijving": {
          "type": ["string", "null"],
          "description": "Short description if present."
        },
        "openbaarmakingsdatum": {
          "type": "string",
          "format": "date",
          "description": "Publication date (YYYY-MM-DD)."
        },

          "type": ["string", "null"],

          "description": "Display name of publishing organisation."
        },
        "aanbieder": {
          "type": "string",

        },
        "mutatiedatumtijd": {
          "type": ["string", "null"],
          "format": "date-time",
          "description": "Last modification timestamp."
        }
      },
      "additionalProperties": true
    },
    "bestandsgrootte": {
      "type": ["string", "null"],

    },
````

A possible top-level schema for the full search response
(**adapt this once you inspect the real JSON in your project**):

```json
{
  "$schema": "https://json-schema.org/draft/2020-12/schema",
  "$id": "https://schemas.local/open-overheid/search-response.json",
  "title": "OpenOverheidSearchResponse",
  "oneOf": [
    {
      "type": "array",
      "items": { "$ref": "search-hit.json" }
    },
    {
      "type": "object",
      "description": "Wrapper style with embedded hits.",
      "properties": {
        "resultaten": {
          "type": "array",
          "items": { "$ref": "search-hit.json" }
        },
        "_embedded": {
          "type": "object",
          "properties": {
            "openbaarmakingen": {
              "type": "array",
              "items": { "$ref": "search-hit.json" }
            }
          },
          "additionalProperties": true
        },
        "page": {
          "type": "object",
          "description": "Optional pagination info.",
          "properties": {
            "size": { "type": "integer" },
            "totalElements": { "type": "integer" },
            "totalPages": { "type": "integer" },
            "number": { "type": "integer" }
          },
          "additionalProperties": true
        }
      },
      "additionalProperties": true
    }
  ]
}
```
